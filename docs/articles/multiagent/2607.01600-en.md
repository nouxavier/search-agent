# Learnings — *BOUNDARY_SYNC: Measuring Communication-Induced Representational Coupling in Multi-Agent LLM Systems*

> **Zewen Liu** (Qilu Institute of Technology) · arXiv:[2607.01600](https://arxiv.org/abs/2607.01600) · preprint 2026
> **Angle:** build. I read this as the floor plan for a "learn-by-building" project — each method in the paper becomes a system component that pulls in an area of math (information theory, distributions, inferential statistics, opinion dynamics).
> **Scope:** focus on the *measurement apparatus* (JSD → CAF → bootstrap) and the design-guiding findings. Left out the stimulus-domain details (gender-stereotype assessment) and the fine discussion of per-model artifacts, which only matter for exact replication.

---

## TL;DR

When LLMs talk to each other in a multi-agent system, communication tends to **homogenize** their outputs (they echo one another), and the paper builds a ruler to measure it: the **CAF = JSD_condition / JSD_baseline**. The insight that carries the rest is that this coupling is **stateless** — it exists only while the neighbors' opinion is in the prompt and vanishes when it's removed, contradicting the classical models (DeGroot) that predict cumulative convergence. Practical consequence: homogenization is *measurable* and *controllable at the prompt level*.

---

## The question and why it matters

Multi-agent systems put several LLMs together to debate/collaborate, betting on diversity of perspectives. The silent risk: if communication makes the agents converge, you pay for N agents but effectively have 1. Before this work, several papers *suspected* the effect ("herding", "persona collapse", "conformity"), but there was no **standardized measurement protocol** that (a) isolated communication as the cause, (b) put everything on a common scale, and (c) separated a transient effect from a cumulative opinion update.

## The mathematical apparatus (what interests me most)

Interlocking pieces, from raw data to a number with an error bar.

**1. Opinion becomes a probability vector.** Each agent responds with a distribution over 5 categories that sums to 1. This turns "opinion" into a measurable object and avoids comparing free text.

**2. The difference meter — Jensen-Shannon (JSD).** A symmetric, finite version of KL:
- `D_KL(P‖Q) = Σ pᵢ log(pᵢ/qᵢ)` — asymmetric, and blows up if `qᵢ=0`.
- `JSD(P,Q) = ½ D_KL(P‖M) + ½ D_KL(Q‖M)`, with `M = ½(P+Q)`. Between 0 (identical) and 1. Low JSD between the previous strategy and the new response = agent moved little = glued to the consensus.

**3. The star metric — CAF (a ratio).**
`CAF = JSD_cond / JSD_baseline`. Dimensionless. **<1 homogenizes, ≈1 independent, >1 diversifies.** Real example: `0.170/0.208 = 0.816`. The ratio only means something with a **modality-matched baseline** — comparing image against a text baseline misleads because the initial diversity differs.

**4. The coupling mechanism — Beta mixing.** Every 3 rounds the prompt is injected with: `p_blend = w·p_self + (1-w)·p_neighbors`, with `w ~ Beta(3,2)` (mean `3/5 = 0.6` self-weight). Robust to varying to Beta(3,3).

**5. Rigor — bootstrap.** CAF is a ratio of means with N=30; the 95% confidence interval comes from 2000 resamples (ratio-of-means). Effect size is also reported (Cohen's d = 1.30, "large") and Welch's t-test. Jackknife as a bias check (≤0.003).

**6. The comparison model — noisy DeGroot.** `x^(t+1) = W·x^(t)` predicts monotonic cumulative convergence. It serves as a contrast to reveal that the LLM does *not* behave this way.

---

## The five findings

1. **Text homogenizes** (CAF=0.803, d=1.30, p<0.001). Social control: replacing the neighbors' consensus with irrelevant Wikipedia text eliminates the effect → it's *social*, not an artifact of a longer prompt.
2. **Image also homogenizes** in a similar proportion (CAF=0.834), but with a larger absolute effect because it starts from more diversity.
3. **Group size flips the direction:** K=5 homogenizes (CAF<1); K=3 diversifies (CAF>1). It's a design parameter, not a confound to control away.
4. **Varies widely across models** (CAF 0.034–0.803), but DeepSeek's near-zero is a format artifact (constrained JSON), not real coupling.
5. **Coupling is stateless** — a sawtooth pattern in the per-round JSD, with no accumulation. Continuous consensus produces monotonic convergence to ~0; intermittent consensus produces the reversible sawtooth. This is the key evidence against DeGroot.

## Practical usefulness (why it's worth building this)

- **Diversity thermometer:** the CAF is a grade for your system. Well below 1 = agents became echoes, multi-agent architecture wasted.
- **Design knobs:** want diversity (brainstorm) → small groups (K=3) and hide the consensus during ideation. Want consensus (fact-checking) → larger groups and keep the consensus circulating.
- **Reversibility as a safety net:** since the effect disappears when you remove the consensus from the prompt, you can toggle convergence on/off at will.
- **Model choice matters:** susceptibility to coupling is a property of the model — worth benchmarking beforehand.

## What to apply in practice to my agents

Direct translations of the findings into engineering decisions — things I can do in my system today.

**Instrumentation (measure before optimizing)**
- **Add the CAF as a system-health metric.** Before responding, each agent emits a structured distribution (even if internally); log the JSD between rounds and the ratio against a no-communication baseline. If the CAF drops below ~0.9, my agents are becoming echoes and the multi-agent architecture is being wasted.
- **Always run a "no communication" ablation.** It's the only way to know whether the convergence came from consensus or would happen naturally. Without a baseline, the number means nothing.
- **Force measurable outputs where it matters.** Even in free-text pipelines, ask for a parallel structured verdict (score/label/distribution) just to be able to measure coupling.

**Flow design (use the right knobs)**
- **Ideation phase = hide the consensus.** If I want divergence (brainstorm, generate hypotheses, ensemble of solutions), do NOT show the agents what the others said during generation. Merge only in the convergence phase. The effect is reversible, so this works in practice.
- **Decision phase = show the consensus and use a larger group.** For fact-checking or yes/no decisions, keep the consensus circulating and use K≥5 to force useful homogenization.
- **Choose K on purpose.** K=3 tends to diversify, K=5 to homogenize. Group size is a design lever, not a detail — set it by the task's objective.

**Prompt and weights**
- **Control the neighbor weight explicitly.** The instruction "weight your output at X% and the consensus at Y%" works; start at 60/40 (self/neighbor). The result is robust to moderate variation, so don't waste time tuning that number.
- **Preserve baseline diversity with distinct personas/temperatures.** Identical agents zero out diversity artificially. Varying persona and temperature per agent gives an honest starting point.
- **Remember that temperature = concentration of the distribution.** If I want more output diversity, raising the temperature has an effect analogous to removing the consensus from the prompt.

**Reliability and cost**
- **No implicit consensus "state".** Since coupling is stateless, I don't count on agents to "remember" a previous convergence — if I need it to persist, I have to re-inject the consensus at every step (continuous consensus → monotonic convergence).
- **Benchmark the model before choosing the backbone.** Susceptibility to coupling varies widely across models; run the CAF test on the candidate before committing.
- **Beware constrained formats (JSON-only, reasoning disabled).** They can produce *artificial* homogenization (agents copying structure) that looks like consensus but isn't. If you see a CAF near zero, suspect an artifact before celebrating "consensus".
- **Start small and cache.** Small N + response caching to validate the pipeline; only scale once the measurement is correct.

## Honest caveats (from the author himself)

- **Self-anchoring:** the prompt includes the agent's own previous output, so part of the drop in CAF may not be social. A "self-only" control is missing → the values are an *upper bound* on social coupling.
- **Group size × prompt size** are confounded (more neighbors = larger prompt).
- **Single domain** (gender stereotypes); may differ for reasoning/code/creative writing.
- **Some results are preliminary** (K=3 without bootstrap CI; N=5/N=10 pilots).

## Hooks for my project (learn-by-building)

The plan we derived: rebuild a mini-BOUNDARY_SYNC from scratch, each phase pulling in an area of math.
- **Phase 1** → implement entropy/KL/JSD by hand (information theory).
- **Phase 2** → simulator with Dirichlet + Beta + DeGroot (distributions and opinion dynamics), zero cost.
- **Phase 3** → CAF and matched baselines.
- **Phase 4** → bootstrap, Cohen's d, Welch (inferential statistics) — the transferable core.
- **Phase 5** → plug in real GPT/Claude.
- **Phase 6** → reproduce the 3 core findings; contrast the sawtooth (LLM) vs. monotonic convergence (simulated DeGroot).
- **Phase 7** → extensions: game theory, graphs, MARL.
